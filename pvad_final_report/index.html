GRU 全面壓制 CAM++：F1 +0.065、TAR +0.047、FAR −0.090。 GRU 只佔 CAM++ 2.6% 的模型大小（0.74 MB vs 28 MB），但因為是 per-target 端到端訓練， 直接學「目標 vs 非目標」的判別，遠勝 CAM++ 的通用 speaker embedding 距離。

Recipe — 兩模型的計算規則

# Per 0.5s segment, both use per-file z-score + threshold sweep CAM++ (原 A8 Triple Ensemble) cos = cos(emb, target_centroid) # embedding 距離 asnorm = (cos − cohort.mean()) / cohort.std() # 對 cohort 標準化 discr = cos − max(cohort_cos) # 相對 cohort 邊距 score = zscore(cos + asnorm + discr) # pred ≥ -0.15 GRU (SD ConvNet+BiGRU, per-target 訓練) p[4] = softmax( ConvNet(LPS+Δ+Δ²) → BiGRU → MarginFC ) # 4-class score = zscore( mean(p[1] − (p[0] + p[2] + p[3])) ) # pred ≥ -0.05

各模型怎麼選

CAM++ (A8)

28 MB · 6.9M
F1 0.858 · TAR 0.88 · FAR 0.170

選它如果：使用者要 plug-and-play（只需 192-d enrollment centroid 768B 即可，無需訓練），場景是任意新使用者。

限制：FAR=0.17 未達 0.20 邊緣達標但較高；spk7 上 FAR=0.20 偏高。

GRU (SD)

0.74 MB · 189K
F1 0.924 · TAR 0.93 · FAR 0.080

選它如果：固定使用者、手機 / 邊緣裝置、模型大小是首要。三項目標 (F1≥0.85 / TAR≥0.90 / FAR≤0.20) 全達。

限制：每個新使用者必須重新訓練一顆 .pth；fan noise 上 F1=0.876 是相對弱項。

附錄 — 把兩者合成會怎樣 (ensemble 上界參考)

CAM++ × GRU 4-vec Ensemble： combo = 0.5·z(CAM_cos) + 0.5·z(CAM_discr) + 2.0·z(GRU) · thr=0.05
結果：F1 = 0.9386（比 GRU alone 再 +0.015）、TAR = 0.932、FAR = 0.054（比 GRU 再 −32%）
重點：F1 邊際提升小，但 FAR 大幅下降；fan noise 從 GRU 0.876 → ENS 0.910（+0.033，最大贏家）。

Ensemble 的價值不在 F1，而在「補位 fan noise + 削 FAR」。如果應用場景容忍 FAR=0.08，純 GRU 已足夠且省 96% 算力 / 模型大小。
