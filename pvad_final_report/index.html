優化歷程：F1 從 0.73 到 0.94

關鍵突破點

1. Fbank Bug 修正 (+0.014)
HANDOVER 的 baseline 用了錯誤的 fbank：32ms FFT 全長視窗 + 無 CMN。修正為 Kaldi 標準 (25ms Povey + preemphasis 0.97 + CMN) 後，centroid cos(spk4,spk7) 從 0.62 (異常) 降到 0.095 (正常)。
2. 對齊 0.5s 邊界 (+0.054)
最大單一改進。fixed_0.5s 已知段邊界，避免滑動視窗跨段汙染。每段中心抽 0.3s chunk → F1 0.74 → 0.79。
3. Multi-chunk + Ensemble (+0.030)
每段抽 3 個錯位 chunk 取平均 → 0.82。再加 AS-norm + Discriminative 標準化合成 → A8 ensemble 0.85。
4. SNR-aware threshold (停滯)
測 7 種自適應策略，效果與 A8 相當。原因是 A8 的 z-score 已隱含 per-file 適配。
★ 5. SD ConvNet+BiGRU Ensemble (+0.090)
引入 archive/inference 內 per-target 訓練的 4-class 模型 (189K params, 0.74MB)。SD 提供 per-target 判別力，A8 提供 speaker-agnostic 距離；兩者誤分模式正交，per-file z-score 加權合成後 F1=0.9386。spk7 從 0.84→0.92、fan noise 從 0.78→0.91。
★ 6. 4-vector ensemble 解構 (+0.001)
把 A8 內三個 score (cos / asnorm / discr) 解開後 5×5×5×5 sweep。最佳 0.5·cos + 0·asnorm + 0.5·discr + 2.0·sd。Top-6 全部丟掉 asnorm — SD 已隱含類似標準化。
7. P2 結構假設 (+0.140)
強制 target/非 target 嚴格交錯，整檔選 H1 (偶=target) 或 H2 (奇=target) 較高分的方向。F1=0.99 但僅適用嚴格交錯場景。

★ SD × A8 4-vec Ensemble 部署 spec (新最佳)

[Input] 16kHz mono audio stream ────── Branch A: CAM++ A8 (2 inferences/sec) ────── [A1] Buffer 0.5s window (aligned to segment boundary) [A2] Extract 3 chunks @ offsets {0.05, 0.10, 0.15}s, each 0.3s [A3] Fbank (Kaldi: Povey 25ms + preemph 0.97 + CMN) [A4] CAM++ batch inference: input shape [3, T, 80] → [3, 192] [A5] Per-chunk scores: cos_t = cos(emb, target_centroid) discr = cos_t - cohort.max() # asnorm dropped! [A6] Average 3 chunks → per-segment cos_seg, discr_seg [A7] Per-file z-score → cos_z, discr_z ────── Branch B: SD ConvNet+BiGRU (continuous, 4ms hop) ────── [B1] Extract LPS + Δ + Δ² (n_fft=256, hop=64, 129 freq) [B2] SD model inference: [1, 3, 129, T] → [1, T, 4] logits [B3] Softmax → probs[:, 4] [B4] Continuous score: p[1] - (p[0]+p[2]+p[3]) [B5] Mean over each 0.5s segment → sd_seg [B6] Per-file z-score → sd_z ────── Fusion ────── [F1] combo = 0.5·cos_z + 0.5·discr_z + 2.0·sd_z [F2] Threshold combo ≥ 0.05 → binary per-segment decision [F3] Duplicate to 5 × 0.1s output slots [Output] 10 binary labels/sec, latency ≈ 500ms [Compute] ~50ms CPU per 500ms (CAM++ 30 + SD 15 + fusion 5) [Memory] CAM++ 28MB + cohort 150KB + SD model 0.74MB + target 768B Total: ~29 MB per user (SD must be trained per-target) [Metrics] F1=0.9386 TAR=0.932 FAR=0.054 (overall) no_noise=0.989 crowd=0.961 fan=0.910 (最弱項)

Fallback: A8-alone deployment (適用任意新使用者)

[Input] 16kHz mono audio stream [Step 1] Buffer 0.5s window (aligned to segment boundary) [Step 2] Extract 3 chunks @ offsets {0.05, 0.10, 0.15}s, each 0.3s [Step 3] Fbank (Kaldi: Povey 25ms + preemph 0.97 + CMN) [Step 4] CAM++ batch inference: input shape [1×3, T, 80] [Step 5] Per-chunk: cos(emb, target_centroid), cos(emb, cohort) [Step 6] Per-chunk scores: cos_t / asnorm / discr [Step 7] Average 3 chunks per score, standardize per file (z-score) [Step 8] Ensemble = z(cos) + z(asnorm) + z(discr) [Step 9] Threshold (calibrated ≈ -0.7) → binary decision [Step 10] Duplicate decision to 5 × 0.1s output slots [Output] 10 binary labels/sec, latency ≈ 500ms [Compute] ~30ms CPU per 500ms (real-time × 17) [Memory] CAM++ (28MB) + cohort matrix (150KB) + target centroid (768B) [Metrics] F1=0.85 TAR=0.86 FAR=0.17

SD ConvNet+BiGRU 特性

架構：3-channel LPS+Δ+Δ² → Conv2D×3 → BiGRU → MarginLinear (4 classes: BG / Main target / SecA / SecB)。
輸入：n_fft=256, hop=64 (4ms), 129 freq bins
權重：archive/inference/best_4class_model_{4,7}{,_0.05}.pth，per-target 訓練（model_4 處理 spk4 檔，model_7 處理 spk7 檔）
二元化：is_main = p[1] > p[0]+p[2]+p[3]（softmax後）
互補性是 ensemble 成功關鍵：A8 對 spk7 (女) 拖後腿（0.84），SD 對 fan noise 拖後腿（0.78）。Ensemble 後 spk7 跳到 0.92、fan 跳到 0.91 — 兩者誤分模式正交，per-file z-score 標準化讓 score scale 一致。

為何 CAM++ 勝出

1. 訓練語言匹配
CAM++ 用 3D-Speaker 200k 中文 speaker 訓練 (CNCeleb + 內部)。WeSpeaker ResNet34 用 VoxCeleb (英文為主)，跨語言能力較弱。
2. CAM 結構優勢
Context-Aware Masking 模組對短 chunk (0.3s) 嵌入更穩定。Paper benchmark 顯示 CAM++ 在 short utterance 場景優於 ECAPA。
3. ECAPA cos(spk4,spk7)=0.53 是崩潰訊號
正常 speaker embedding 對不同人應 cos ≈ 0.05–0.15。0.53 表示 embedding 根本沒區分這兩人。原因可能：fbank 規格 (CMN/preemph) 不對；或 ONNX 是某 baseline 版非 production。

已下載驗證的其他模型

CAM++ (current, sherpa-onnx 200k Chinese): 28 MB, 192-dim F1=0.85 ★ funasr/campplus (HuggingFace .bin): 27 MB, 192-dim 同架構 (規格一致，未測) WeSpeaker ResNet34: 25 MB, 256-dim F1=0.77 ECAPA-TDNN (small, 575KB): 575 KB, 192-dim F1=0.67 (崩潰) AXERA ECAPA (3D-Speaker, 79MB): 79 MB, 192-dim fixed [1,360,80] 輸入 (不適合 0.3s) AXERA Res2Net v2 (3D-Speaker, 68MB): 68 MB, 192-dim fixed [1,1,360,80] 輸入 (不適合 0.3s)
結論：本地可用 CAM++ 已是最強。AXERA 版固定 360 frame 輸入不適合短 chunk，僅供研究參考。下一輪可探索：3D-Speaker ERes2Net、sherpa-onnx 其他 CAM++ 變種、或 fine-tuned 版本。

Input: LPS (Log Power Spectrum) - n_fft=256, hop=64 (4ms), Hamming window - 3 channels: [LPS, delta, delta2] - 129 freq bins Network: Conv2d(3→32, (129,5)) → GroupNorm → LeakyReLU Conv2d(32→64, (1,5)) → GroupNorm → LeakyReLU Conv2d(64→128, (1,5)) → GroupNorm → LeakyReLU → permute → squeeze freq → [B, T, 128] BiGRU(128→64×2) → LayerNorm → [B, T, 128] MarginLinear(128, 4) # ArcFace-style margin layer Output: per-frame 4-class logits class 0: silence class 1: target class 2: same-gender non-target class 3: diff-gender non-target
