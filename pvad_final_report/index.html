優化歷程：F1 從 0.73 到 0.85

關鍵突破點

1. Fbank Bug 修正 (+0.014)
HANDOVER 的 baseline 用了錯誤的 fbank：32ms FFT 全長視窗 + 無 CMN。修正為 Kaldi 標準 (25ms Povey + preemphasis 0.97 + CMN) 後，centroid cos(spk4,spk7) 從 0.62 (異常) 降到 0.095 (正常)。
2. 對齊 0.5s 邊界 (+0.054)
最大單一改進。fixed_0.5s 已知段邊界，避免滑動視窗跨段汙染。每段中心抽 0.3s chunk → F1 0.74 → 0.79。
3. Multi-chunk + Ensemble (+0.030)
每段抽 3 個錯位 chunk 取平均 → 0.82。再加 AS-norm + Discriminative 標準化合成 → A8 ensemble 0.85。
4. SNR-aware threshold (停滯)
測 7 種自適應策略，效果與 A8 相當。原因是 A8 的 z-score 已隱含 per-file 適配。
5. P2 結構假設 (+0.140)
強制 target/非 target 嚴格交錯，整檔選 H1 (偶=target) 或 H2 (奇=target) 較高分的方向。F1=0.99 但僅適用嚴格交錯場景。

Hybrid 部署 spec

[Input] 16kHz mono audio stream [Step 1] Buffer 0.5s window (aligned to segment boundary) [Step 2] Extract 3 chunks @ offsets {0.05, 0.10, 0.15}s, each 0.3s [Step 3] Fbank (Kaldi: Povey 25ms + preemph 0.97 + CMN) [Step 4] CAM++ batch inference: input shape [1×3, T, 80] [Step 5] Per-chunk: cos(emb, target_centroid), cos(emb, cohort) - target centroid: 192-d from user enrollment - cohort: 200 AISHELL-1 speaker centroids (shipped, ~150KB) [Step 6] Per-chunk scores: - cos_t = cos(emb, target) - asnorm = (cos_t - cohort.mean()) / cohort.std() - discr = cos_t - cohort.max() [Step 7] Average 3 chunks per score, standardize per file (z-score) [Step 8] Ensemble = z(cos) + z(asnorm) + z(discr) [Step 9] Threshold (calibrated ≈ -0.7) → binary decision [Step 10] Duplicate decision to 5 × 0.1s output slots [Output] 10 binary labels/sec, latency ≈ 500ms [Compute] ~30ms CPU per 500ms (real-time × 17) [Memory] CAM++ (28MB) + cohort matrix (150KB) + target centroid (768B)

天花板分析

為何 TAR≥0.90 + FAR≤0.20 同時不可達

從 SNR-aware 實驗的數據看，TAR-FAR 操作點分佈如下：
FAR約束 最大 TAR F1 ≤0.05 0.30 0.46 ≤0.10 0.55 0.66 ≤0.15 0.75 0.78 ≤0.20 0.87 0.84 ← 目標 TAR 0.90 不可達 ≤0.25 0.91 0.85 ← TAR=0.90 在 FAR=0.22 處實現 ≤0.30 0.95 0.85
物理上限：在 CAM++ 0.3s chunk-level 嵌入下，TAR=0.90 必須 FAR≥0.22。要 TAR≥0.90 同時 FAR≤0.20，必須換更強嵌入或前端去噪。

突破上限的三條路

路徑 A

換更強模型

預期 F1 +0.05~0.10

WeSpeaker ResNet34 (本地有權重)、ECAPA-TDNN、3D-Speaker 200k 中文模型 (Hugging Face)。CAM++ 是輕量版，更大的模型在 0.3s chunk 上有更穩定嵌入。

路徑 B

Fine-tune CAM++

預期 F1 +0.10+

用 AISHELL + 噪音增強做 PVAD-specific fine-tuning。學習對短 chunk + noisy 條件的特化 embedding。需要訓練架構與 GPU。

路徑 C

Speech Enhancement 前端

noisy F1 +0.05~0.10

用 DTLN / GTCRN / TSE 等去噪模型先處理音檔，再算嵌入。Clean 條件不變，noisy 條件大幅改善。增加延遲與計算量。

不投入路徑 A/B/C 時的最終水平

A8 Triple Ensemble 是真實天花板
F1=0.85, TAR=0.86, FAR=0.17 — 這就是 CAM++ + 不微調 + 不換模型 配置下的最佳成果。
