## 4. 核心發現

### 結果摘要

在所有 72 個實驗條件中（6 SNR × 2 噪音 × 6 分類器），僅 2 個條件出現準確率下降：

1. **Utterance-level, Babble -5dB, SVM-RBF**: Accuracy = 0.923, F1 = 0.933
2. **Frame-level 0.5s, Babble -5dB, KNN (K=3)**: Accuracy = 0.923, F1 = 0.933

其餘所有條件均達到 **100% 準確率**。

### 為什麼噪音影響如此小？

1. **Embedding 空間的巨大分離度**: 原始實驗顯示 hsuan 的 cosine similarity 集中在 0.88–0.97，
   非目標在 0.13–0.25，中間有約 0.6 的間隔。即使 -5dB SNR 的噪音也無法將 embedding
   推移超過這個間隔。

2. **WeSpeaker 的噪音魯棒性**: WeSpeaker ResNet34 在 VoxCeleb 上訓練時已包含各種
   噪音增強，模型對加性噪音有內建的魯棒性。

3. **說話者差異太大**: 只有 3 位說話者，且聲學特徵差異明顯。在更多、更相似的說話者
   條件下，噪音的影響會顯著增大。

4. **Simulated babble 的侷限**: 本實驗的 babble noise 是用正弦波合成模擬的，
   不如真實多人語音的干擾效果強。真實 babble noise（如 MUSAN 資料集）會更具挑戰性。

### 真正需要的挑戰條件

要讓分類器之間出現有意義的差異，需要：

1. **更多、更相似的說話者**: 同性別、相近年齡的說話者對，使 embedding 空間中的
   clusters 更接近。

2. **真實噪音**: 使用 MUSAN、DEMAND 等噪音資料集中的真實環境噪音和多人語音混合。

3. **混合語音（Overlapped Speech）**: 目標與干擾說話者同時說話，embedding 被污染。

4. **極短窗口**: 0.1s–0.2s 的窗口長度，讓 embedding 品質大幅下降。

5. **房間混響（Reverberation）**: 加入 RIR (Room Impulse Response) 模擬遠場效果。

## 5. 實用建議

1. **當前場景下維持 Cosine Similarity**: 在 3 說話者的場景中，
   即使在最嚴苛的噪音條件下，cosine similarity 也保持 100%。
   沒有理由引入更複雜的分類器。

2. **擴展測試條件**: 建議收集更多說話者的資料（特別是與 hsuan 聲學特徵相似的說話者），
   並使用真實噪音資料集重新測試。

3. **pVAD 的真正瓶頸**: 不在分類器選擇，而在串流場景下的即時性、
   混合語音處理、以及 enrollment 品質。
