關鍵發現

① Fbank bug 是過去評估失準的根源
HANDOVER 的 F1=0.7255 基線是用錯誤 fbank 跑出。換成 Kaldi 標準 (25ms povey + preemph 0.97 + CMN) 後，centroid cos(spk4,spk7) 從 0.62 降到 0.095，所有後續方法才有意義。
② 對齊抽取 (利用 0.5s 邊界先驗) 是最大改進
從 sliding baseline F1=0.74 跳到 aligned F1=0.79（A0 single）→ multi-chunk F1=0.82 (A3)。0.3s 視窗對 0.5s 段而言，邊界 ±0.1s 是污染區，避開後 F1 跳 +0.08。
③ Ensemble (cos + AS-norm + Discriminative) 比單一方法穩
A8 ensemble F1=0.85，三種方法錯誤模式不同：baseline 對 threshold 敏感、AS-norm 對 cohort 偏置敏感、discriminative 對 max cohort 敏感。融合互補。
④ Force-alternation (P2) 突破天花板
利用 fixed_0.5s「嚴格交錯」結構約束。從 ensemble 的 F1=0.85 (chunk-level decision) 跳到 P2 的 F1=0.99 (file-level alternation)。代價：僅適用嚴格交錯場景。
⑤ Hangover fill 失敗，Smart fill 持平
P1 hangover 太激進（任何 0,1,0 都填 1 → FAR 爆 0.93）。P3 smart fill（要分數接近 threshold 才填）跟 raw ensemble 持平。對 ensemble 後處理而言，原始預測已經接近最佳。

部署建議

場景 A: 已知嚴格交錯 (如 fixed_0.5s)
→ 用 P2_alternate。F1=0.99 全部達標。
場景 B: 結構不確定的通用 PVAD
→ 用 A8 / raw_base_heavy ensemble。F1=0.85, TAR=0.85, FAR=0.16。差 TAR -0.05 到目標。
場景 C: 要更高 TAR 但能放寬 FAR
→ 降低 ensemble threshold 取 raw_equal F1-best 點。或考慮換 ECAPA-TDNN / 3D-Speaker 200k 模型。
