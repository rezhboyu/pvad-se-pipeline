# DTLN 降噪效果評估報告 — 真實中文語音

**日期**: 2026-03-20
**模型**: DTLN (dtln_1.onnx + dtln_2.onnx)
**取樣率**: 16kHz
**測試語音**: MAT 資料集（hsuan、0911636193、FEMH 各一段）

---

## 實驗設計

對三位不同說話者的中文語音，分別加入以下噪音條件後用 DTLN 進行降噪：

- 白噪音 (White): SNR 20dB / 10dB / 5dB / 0dB
- 粉紅噪音 (Pink): SNR 10dB / 5dB
- Babble noise (其他說話者語音疊加): SNR 10dB / 5dB

共 3 × 8 = 24 組測試。

**評估指標**: SNR improvement (dB)、PESQ (wb)、STOI

---

## 核心發現：DTLN 模型存在嚴重的訊號衰減問題

DTLN 模型在所有測試條件下都將輸出 RMS 壓縮至輸入的約 25%。這意味著即便對乾淨語音，模型也會引入約 -12dB 的增益衰減。這導致 SNR 指標在大多數情況下不僅沒有改善，反而大幅惡化。

**乾淨語音測試**: 將乾淨語音直接過 DTLN 後，輸出 RMS 僅為原始的 25%，失真 SNR 約 0.7~4 dB。

## 分析

1. **訊號衰減是主因**: DTLN 在所有條件下都將訊號能量壓縮到 ~25%，這嚴重拖累了所有客觀指標。
2. **高 SNR 反而更差**: 當原始訊號品質較好（SNR=20dB）時，DTLN 引入的失真反而比噪音更大，導致指標大幅惡化。
3. **僅在極端噪音下有效**: 只有 SNR ≤ 0dB 時，DTLN 的降噪效果才能蓋過其自身引入的失真。
4. **Babble noise 最差**: 模型對其他說話者語音型噪音的處理效果最差，PESQ 和 STOI 在 babble 條件下一律惡化。
5. **說話者依賴性**: FEMH（RMS 最低，0.02）的相對改善最大，hsuan（RMS 最高，0.13）最差。

---

## 可能原因與建議

1. **模型未針對中文語音訓練**: 預訓練 DTLN 主要用英文語音，中文語音的頻譜分布不同。
2. **增益校準問題**: 模型輸出能量過低，可能需要在後端加一個增益補償步驟。
3. **ONNX 導出問題**: 確認 ONNX 模型是否與原始 TensorFlow 模型的輸出一致。
4. **建議下一步**:
   - 嘗試在 DTLN 輸出後加 RMS 正規化（matching clean 的 RMS）
   - 測試其他 SE 模型（如 FullSubNet、DCCRN）
   - 用中文語音資料 fine-tune DTLN
