FP 高 cos 微片段驗證

每個 FP 段裡 cos 最高的 0.3s 窗口（pad 到 0.5s 方便聽）。如果你聽到的是 target 聲音 → boundary bleed；如果是不同的聲音 → CAM++ chunk-level 偶發雜訊（這是 0.3s 嵌入本身的限制）。

觀察重點：
🟡 END/START boundary: 高 cos 在 FP 段的最邊緣，可能因為 GT 標註精度（1ms = 16 samples 誤差）讓 target 聲音稍微 bleed 進來。
🔴 MID: 高 cos 在 FP 段中間（離兩邊都遠），不可能是 boundary bleed。**是 CAM++ chunk-level 嵌入的真實雜訊**。

FP#3 (target=spk4, label=3) — 關鍵 case

peak @ 0.75-1.05s (MID, 離邊界 0.5s+) cos=0.322MID 確定無 bleed
這個最關鍵：FP 是 1.57s 長，這個高 cos 0.3s 在中間（0.75-1.05s），離兩邊都超過 0.5s。不可能是 boundary bleed。如果這聽起來像 User1 → CAM++ 真的把這個非目標誤判；如果不像 → CAM++ chunk-level 嵌入有雜訊（0.3s 太短）。

結論依據

聽完後請告訴我：
  1. FP#3 MID 那個（0.75-1.05s, cos=0.322）— 聽起來像 User1 嗎？
  2. END boundary 的（FP#1, #2, #4 的 END）— 有沒有聽到 target 聲音的痕跡？
  3. MID 的（FP#1 0.25-0.55s, FP#2 0.70-1.00s）— 完全不像 target？

判定：
A. 全部都不像 target → 模型誤判，是 0.3s 嵌入雜訊（已知問題，靠 smoothing 緩解）
B. END 的有，MID 的不像 → boundary bleed 是主因（修 GT 精度可解）
C. MID 的也像 target → 有真實的聲學重疊，或 mix 時有 bug（需查代碼）
