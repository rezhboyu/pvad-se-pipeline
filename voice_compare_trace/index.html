FP 段落 + 完整上下文對比

每個 FP 提供三個版本：① pure = 純非目標段（GT 標的範圍）；② wide = FP±1 秒（含相鄰目標段，會聽到 target 聲音）；③ full = 完整 source 檔（看 GT JSON 對應）。先聽 pure，那才是模型真正被誤判的內容。

判讀

① pure 才是「真正的非目標段」 — 是 GT label 標為 label 2 或 3 的純非目標說話人 audio。如果這個版本聽起來跟 target enrollment 完全不像 → 模型不該誤判，是 chunk-level cosine 失準。

② wide 包含相鄰 target 段，所以一定聽得到 target 聲音 — 但那不是 FP 內容。

③ full source 是完整的混音檔（30+ 秒），可以聽出整個交錯 target 和非目標的結構。

FP#4 特別注意：target segment 只有 0.20-0.23 秒，這種極短切換對 0.3s 窗口的 PVAD 是極端 case，邊界 bleed 不可避免。
